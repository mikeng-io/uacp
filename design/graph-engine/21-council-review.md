# Council Review — six lenses (2026-06-20)

Six independent Opus reviewers read the full bundle. **Overall: CONCERNS** — zero findings kill the
core idea; several are PLAN-exit gating fixes. The serialization discipline (typed edges, provenance,
the two missing keys `scope_item.id`+`work_unit.derives_from`, the closure checks, files-as-truth) is
validated by all lenses. The findings target the wrapping (substrate, packaging, enforcement, scope).

| Lens | Verdict | Single most important change |
|---|---|---|
| Architecture/abstraction | CONCERNS | Index port = **query-only**; no write-veto / closure copy (fix CQRS leak) |
| Data integrity/correctness | CONCERNS | Make `_index.yaml` **fully derived** from node frontmatter (+ `index-consistency` check) |
| Storage/tech-selection | CONCERNS | Re-scope claims + bake-off gate to the **filtered** query (3 axes) |
| Governance/security | CONCERNS | Make **validate-on-write real + unbypassable**; Guardian must block raw writes to manifest paths |
| Devil's advocate | CONCERNS (→FAIL as-is) | **Cut the substrate from v1**; ship two keys + in-memory projector + closure checks |
| Integration/fit | CONCERNS (PASS-leaning) | `scope_item` is a real migration (not "add a key") — needs a compat shim; rename the slice |

## Convergent themes (multiple lenses)

**T1 — HIGH — `_index.yaml` is an un-reconciled SECOND source of truth → silent-drop + write race.**
Each edge is serialized 3×: child frontmatter `derives_from`, `_index.yaml.edges`, and `coverage`. No
authority declared; a monolithic `_index.yaml` rewrite reintroduces the exact blast radius D3 killed,
one level up; concurrent `create_*` calls race on it (lost update → orphan/phantom).
*(Integrity F1/F4, Governance F5.)* **Fix:** child node frontmatter is canonical; `_index.yaml`
edges/members/coverage are **derived**, regenerated by `uacp-fmt`, never hand-merged → commutative,
race-free. Add `index-consistency` closure check (aggregate must reproduce from members, else BLOCK).

**T2 — HIGH — Read-model boundary leak (CQRS).** Flows give the Index engine a write-veto (DELETE
returns BLOCK) and a copy of closure (edit/supersede re-check) — the overlap D17 claims is eliminated.
*(Architecture F1.)* **Fix:** Index port answers *questions*, side-effect-free; the *decision* (BLOCK,
phase-advance) belongs to the Manifest engine / Guardian / Heartgate. Make sync one-directional.

**T3 — HIGH — Existence ≠ correctness; provenance is forgeable.** Closure proves edge *topology*, not
that an `asserted` edge points at the *right* node; `provenance` is a free enum a producer can forge
(`relates_to` with `provenance: derived`; a guessed `derives_from` as `asserted`). Post-transition
edit/supersede re-points `asserted` edges with no council re-review. *(Integrity F2/F6/F7, Gov F2.)*
**Fix:** per-`rel_type` provenance constraint in `uacp-schema` (lint rejects violations); `parsed`
edges must be SCIP-reproducible (`forged-parsed` check); any change to an `asserted` edge's dst
re-triggers council; add `contradicted` check (a `pass` assessment whose evidence is `fail`/`violated`).

**T4 — HIGH — Stale-index read at a governance gate.** Heartgate queries the DB for closure with no
watermark precondition; the FS→DB step isn't transactional, so a gate can PASS on a stale graph; the
gitignored index is hand-editable (not in Guardian's protected set). *(Integrity F3, Governance F3.)*
**Fix:** watermark is a **precondition inside** the closure query — `index.watermark != fs.hash` ⇒
rebuild-then-answer or return STALE; Heartgate treats STALE as BLOCK. Protect the index path from raw
writes.

**T5 — HIGH — Enforcement is aspirational (present-tense for unbuilt machinery).** "Guardian invokes
uacp-lint on every write" does not exist — the real Guardian checks tool/path/context, never content;
`config/state.yaml` says runtime enforcement is "not yet implemented." *(Governance F1, Integration F1
confirms the permissive write.)* **Fix:** reword as net-new Slice-1 work; the entity-writer must be the
**only** path that writes a manifest file, and Guardian must **block raw writes** (Write/Edit/
`uacp_artifact_write` blob) to manifest paths — else validate-on-write is advisory.

**T6 — HIGH — Slice 1 is over-scoped; the substrate is premature for v1.** Slice 1 bundles ~5
independently-council-gated projects (uacp-schema, uacp-lint/fmt, entity re-layout, writer promotion,
SQLite projection). The data is "tens–hundreds of nodes/run" — fits in RAM; an in-memory dict projector
(~100–200 lines) running the 5 set-algebra closure checks delivers the whole v1 thesis. The Index port
abstracts over exactly one viable adapter (speculative generality). *(Devil F1/F2/F5/F8, Storage F4,
Integration F2/F3.)* **Fix:** Slice 1 = two keys + in-memory projector + 5 closure checks on **today's**
fixtures + derived `_index.yaml` + real validate-on-write. **Defer:** Index port, SQLite substrate,
schema/lint/fmt as separate packages, constraints/metrics plane, code plane — extract them when real
usage demands. `scope_item` migration needs a legacy-string→id compat shim in the same slice.

**T7 — MEDIUM — Stale contract nodes / facts.** `14-projection-engine` still presents the two-store
SQLite+LanceDB model (contradicts D16 single-DB that `19-storage-summary` adopts); D7 says negative
space is "Slice 2" (everywhere else Slice 3); Kùzu/Cozo status stale (LadybugDB ≈v0.17 + a live Vela
fork). *(Devil F3, Architecture F5, Storage F3.)* **Fix:** reconcile 14 to D16; move D11/D13/D15 to a
"superseded" appendix; amend D7→Slice 3; refresh the bake-off facts.

**T8 — MEDIUM — Quality-evidence overstated.** "RAG quality near-flat across recall 0.4→1.0" is not
what the literature says (reranking only plateaus on the high-recall side) and rests on an
uncorroborated arXiv id; the single-DB reasoning is about *unfiltered* recall while production is
plane/run-*filtered* (sqlite-vec's weakest area). *(Storage F1/F2/F5.)* **Fix:** drop the "near-flat"
claim; re-anchor on the self-sufficient "exact = 100% recall ≥ any ANN"; make the LanceDB-retirement
gate 3-axis (filtered recall@k + p95 latency at size + independent rebuild cost). Verify/remove the
arXiv cite before governance.

## Standalone findings (single lens, still real)

- **Supersede orphans cross-plane/cross-run inbound edges** (lesson/constraint/inherits) into a
  `superseded` node — lineage rot, uncaught. Add `stale-reference` check. *(Integrity F5.)*
- **Tombstone/supersede can escape an obligation** (delete an `unverified` work_unit to clear the
  finding) — routes around "evidence must be produced." Add `deleted-with-open-obligation` check;
  closure must read tombstones. *(Governance F4.)*
- **id-minting race** (`wu-2` sequential) → silent identity merge. Use ULID/locked minting +
  `duplicate-id` backstop. *(Integrity F8.)*
- **`code_symbol` identity is a distinct regime** (repo-scoped, parser-resolved, git-lifecycle) — flag
  it; the "engine-minted, run-scoped" rule is governance-plane-only. *(Architecture F7.)*
- **Aggregate-as-consistency-boundary is decorative** — coverage/unverified are cross-aggregate; the
  **run** is the real closure boundary. Rename: aggregate = write/txn boundary, run = consistency
  boundary. *(Architecture F6.)*
- **Two-engine/CQRS framing may over-abstract** — keep the rule (never embed the manifest; reach it by
  traversal), drop the "two heavyweight engines" mandate (it's one writer + one projector). *(Devil F6.)*

## Positive confirmations

  engines registry (5 engines), `oracle/index_build.py` projection pattern, `governed_handlers.py`
  permissive write, `state_machine.py` edges, `SqliteVecStore` stub, `validate_uacp_artifacts.py`

- LanceDB retirement is **correctly gated** (recall bake-off, not unilateral); reranker is
  store-agnostic so the locked subsystem's invariants survive. Must be a `decision-log.md` entry +
  VERIFY gate. *(Integration F4.)*
- New packages are **ADR-0017-clean** (no-authority-mirrors, one-directional boundary, kind taxonomy).
  *(Integration F5.)* Governance path (TRIAGE→PROPOSE→council, no-main-writes) correctly required.

## Remediation plan

1. **Re-scope Slice 1** to the minimal vertical (T6) — proves the thesis, fixes the cheap integrity
   gaps in a small codebase.
2. **Design fixes (do before build):** derived `_index.yaml` (T1), query-only Index/CQRS (T2),
   per-rel_type provenance + forged-parsed + contradicted (T3), watermark-gated closure (T4), real
   unbypassable validate-on-write (T5).
3. **Doc reconciliation:** 14→D16, superseded appendix, D7 slice, stale facts (T7); quality-evidence
   (T8).
4. **New closure checks to add to the design:** `index-consistency`, `provenance-per-rel-type`,
   `forged-parsed`, `contradicted`, `stale-reference`, `deleted-with-open-obligation`, `duplicate-id`.
5. **Defer:** Index port, SQLite/sqlite-vec substrate, schema/lint/fmt packaging, constraints plane,
   code plane — until usage or scale demands (extract-from-working, not build-ahead).
