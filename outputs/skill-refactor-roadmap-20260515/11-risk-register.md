# Risk Register

## R1: Recreating a mega-SOP

Risk: saving everything into one detailed document.

Mitigation: split artifacts by concern and skill; use this directory as the reference pattern.

## R2: Bulk migration loses detail

Risk: trying to refactor all skills at once causes vague generic content.

Mitigation: one skill at a time; stop after each implementation.

## R3: Current UACP artifacts mislead determinations

Risk: existing docs claim things are solved when they are not.

Mitigation: treat current UACP as suspect; use memory/session recall and ground-truth file inspection.

## R4: Over-engineering every skill

Risk: adding schemas/scripts to simple skills unnecessarily.

Mitigation: variants are allowed; each Decision artifact justifies structure.

## R5: Shared references remain a junk drawer

Risk: phase content continues accumulating globally.

Mitigation: shared cleanup happens last and keeps only primitives.

## R6: Implementation starts too early

Risk: patching before Explore/Determine/Decision/Review/Audit.

Mitigation: no implementation without the five planning artifacts for the active skill.

## R7: Self-certification

Risk: using UACP lifecycle/Heartgate as authority for this repair.

Mitigation: use normal git workflow and independent review; UACP lifecycle is not execution substrate here.
