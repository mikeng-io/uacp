# Adaptive PLAN Package Model

## Universal PLAN core
Every selected adaptive PLAN package must cover these concerns or declare N/A with rationale:

1. `work_breakdown`
2. `dependencies`
3. `authority_and_side_effects`
4. `tool_runtime_selection`
5. `artifact_write_surfaces`
6. `verification_strategy`
7. `rollback_recovery`
8. `council_review_topology`
9. `transition_readiness`

## Context-selected modules
Examples:

- `runtime_adapter_plan`
- `guardian_heartgate_plan`
- `schema_validator_plan`
- `documentation_patch_plan`
- `migration_plan`
- `public_private_boundary_plan`
- `multi_agent_dispatch_plan`
- `kanban_coordination_plan`
- `containment_plan`
- `state_transition_plan`

## Selection standard
Select modules based on the work-heart, affected surfaces, side effects, verification difficulty, and runtime/authority boundaries.

## Boundary
PLAN packages make EXECUTE deterministic. They do not implement the mutations themselves.
