# Adaptive Documentation Model

## Rule

Documentation selection is adaptive and context-driven.

Granularity answers: how much rigor, evidence, review pressure, and transition confidence are required?

The work's heart answers: which documents/modules are needed?

## Universal core concerns

- intent
- authority
- scope
- containment
- risk
- verification
- transition
- artifact map

These are concerns, not mandatory filenames.

## Domain-selected modules

Common modules include:

- requirements/spec delta
- design/architecture
- tasks/work packages
- operations/runbook
- policy/governance
- data/state stewardship
- security/compliance
- research/evidence
- communication/change management
- user/decision journal

## Anti-patterns

- fixed OpenSpec-style document quotas
- granularity-only document selection
- YAML-only serious proposals
- mega-docs hiding missing concerns
- weak blanket `not_applicable`
