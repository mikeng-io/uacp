# Doctrine Delta

## Before

UACP PROPOSE could be treated as sufficiently represented by lifecycle YAML records. A corrective patch then risked replacing that with a fixed OpenSpec-like document list.

## After

UACP PROPOSE uses adaptive documentation selection:

- Universal core concerns must be covered for serious work.
- Domain modules are selected by the work's context/heart, not by granularity alone.
- Granularity controls rigor, evidence depth, and gate strictness.
- Selected concerns are recorded in `package-selection.yaml`.
- PROPOSE→PLAN blocks when selected concerns are missing or dishonestly omitted.

## OpenSpec relationship

OpenSpec is pattern evidence for separated proposal/design/task/spec surfaces. UACP does not inherit OpenSpec's coding-centric file set.

## Trustless ACP relationship

Trustless ACP is pattern evidence for Guardian/state/worktree/control-plane discipline. UACP does not inherit Trustless fixed domains, gates, or coding-agent assumptions.
