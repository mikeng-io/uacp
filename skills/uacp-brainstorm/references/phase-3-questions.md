## Phase 3: Ask Clarifying Questions

Ask the minimum questions needed to reduce ambiguity. **One question at a time.**

**Rules:**

- Prefer multiple choice when possible.
- Maximum 3 questions total.
- Stop asking when scope becomes clear enough to sketch approaches.
- Do not ask questions you can already infer from Phase 1–2.

**Typical question sequence:**

**Q1 — Goal clarity** (if goals are ambiguous):
> "What would make this successful?"
> Options: ship a working prototype / improve reliability / reduce cost / unblock another team / explore feasibility

**Q2 — Constraint boundary** (if constraints are unclear):
> "What is a hard limit I should not cross?"
> Examples: cannot change X API, must be done by Y date, cannot introduce new dependencies

**Q3 — Scope preference** (if multiple directions exist):
> "Given these directions, which feels closest to what you want?"
> Options: A) minimal fix, B) medium refactor, C) larger redesign with migration

**Output of this phase:** refined `brainstorm_signals` and enough clarity to sketch approaches.
